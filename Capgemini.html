Introduction¶

Goal¶

The aim of this work is to study the propagation of messages inciting fraudulent behavior in public transports on social media. We focused on twitter which API is quite convenient to retrieve data. The final goal is to construct a classifier able to distinguish a fraudulent tweet. That work is extensible to other social networks such as Facebook.

Methodology¶

A database of 50 000 tweets with selected key words ('attention', 'controleur', 'rer', ...) related to public transport was constructed. Fraudulent tweets reveal what train stations controlling staff is operating in.

One challenge of the project is the class imbalance. There are many more non fraudulent tweets (90%) than fraudulent ones (10%). This impacted the choice of the performance metric: accuracy alone was not sufficient as the classifier could get a 90% accuracy score by only predicting negatives. The F1 score seemed to be more appropriate.

The corpus of 50 000 documents was appropriately processed (correction, stopwords, punctuation, special characters, links, tokenization...) reducing the number of distinct words.

We calculated a TfIdf matrix with the remaining words and performed a K-means algorithm to obtain clusters of tweets: we created a wordcloud for each cluster to derive its topic. We discarded clusters whose wordcloud did not show any relationship to public transport (for example, a cluster contained the word 'contrôleur' but was actually dealing with job offers for 'contrôleur de gestion').

Feature engineering included the creation of features such as Bigrams (association of 2 words) and the introduction of the tweet size feature. Indeed it appeared in a random forests' feature importance that shorter tweets were associated with fraud.

We manually labeled part of the tweets (a few thousands) of the meaningful clusters with 0 (non fraud), 1 (fraud) or 2 (general criticism of public transports) to have a fair sized training set. On this final matrix, we tried to construct a good classifier for tweets labeled 0, 1 or 2. We then used it on unlabeled data: on each trial, we verified and corrected the label predicted if necessary, which provided us with a larger training set (semi-supervised learning).

# Links, punctuation...
def clean(word):
    word=re.sub(r'https?:\/\/.*[\r\n]*', 'link', str(word), flags=re.MULTILINE)
    word=re.sub(r'http?:\/\/.*[\r\n]*', 'link', str(word), flags=re.MULTILINE)
    word=re.sub(r'\n','', str(word),flags=re.MULTILINE)
    word=re.sub(r'\r','', str(word),flags=re.MULTILINE)
    for p in list(punctuation):
        word=word.replace(p,' ')
    return word.lower()
